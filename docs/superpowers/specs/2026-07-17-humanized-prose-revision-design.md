# 真人化正文与定点二稿设计

## 目标

降低《送个外卖，我能签收顾客的异能》等小说的模板化正文感，同时不牺牲既有的设定一致性、章节状态变化和候选稿审核机制。

## 范围

1. 为“都市脑洞”新书自动写入正文与质检两套真人化提示词模板。
2. 将正文提示词从逐项强制验收改为“场景优先、章纲守底线”的写作要求，并将默认正文温度降为 `0.72`。
3. 让质检结果可生成一份独立的“定点二稿”候选版本：只改 3 至 5 个有正文证据的问题，保留其余事实、结构与专名。

## 不在范围内

- 不自动接受或覆盖正式稿。
- 不重写历史候选稿。
- 不修改现有四合院题材的专项规则。
- 不引入多模型评审、向量数据库或全文重构。

## 设计

### 题材级默认模板

`createNovel()` 在新建“都市脑洞”作品时插入两条小说级模板；`getWorkspace()` 会为已存在但缺失这两条模板的都市脑洞作品补齐同名规则：

- 正文模板：近距离视角、动作先于解释、允许沉默/误解/改口、让物件与限制改变选择、禁止总结性旁白和标准问答。
- 质检模板：检查潜台词是否被旁白解释、人物是否同声同气、场景是否按固定问答推进、主角是否没有试错。

它们与现有 `prompt_templates` 机制兼容，用户可以在设置页编辑或删除。

### 正文提示词

保留章纲、锁定设定、能力来源和结尾钩子等硬边界；删除“70% 前必须兑现”“必须写见证者反应”等逐项显性命令。

正文要求调整为：

- 先让人物在具体场景中行动；
- 章节目标、冲突和状态变化必须完成；
- 奖励、见证、伏笔按剧情需要自然显现；
- 动作或环境已经表达的信息不得由旁白再解释；
- 对话允许停顿、改口、抢话、答非所问与未完成表达。

正文请求温度改为 `0.72`。

### 定点二稿

新增 `startTargetedRevision()`：

1. 输入章节与源候选版本。
2. 对源文本运行现有 `semanticQualityCheck()`。
3. 选择分数低于 75 且有正文证据的前 5 个问题；若不足 3 个，则取最低分问题补足。
4. 调用同一正文模型，要求只改与问题相关的段落，不新增人物、设定、事件、奖励或结局。
5. 保存为新的 `ai-revision` 候选版本，仍需人工接受。

若模型未配置，返回可理解的错误；若质检没有可操作问题，不创建候选稿。

### API 与界面

新增 `POST /api/chapters/:id/revisions`，请求体接受 `{ sourceVersionId }`。

候选版本列表为 AI 候选稿显示“生成定点二稿”按钮；请求中或源版本不是 AI 候选稿时禁用该按钮。完成后刷新版本列表，用户可并排比较并手动接受。

## 验证

- 都市脑洞新书获得两条真人化模板，其他题材不受影响。
- 正文预览不再包含“70%进度前兑现”或“明确写出见证者反应”，并包含场景化规则。
- 定点二稿在没有供应商时失败但不创建版本。
- 使用模型请求的测试桩时，二稿保存为 `ai-revision`，请求中携带源文本和三至五项带证据的问题。
