# 手动保存工作稿为正稿

## 目标

允许作者将章节编辑器中的当前工作稿直接保存为正式正文，不需要先经过 AI 候选稿流程。

## 方案

在章节编辑器顶部新增“保存为正稿”操作。它会：

1. 先保存当前未落盘的工作稿内容。
2. 调用现有章节版本接口创建一个标记为“手动正稿”的不可变版本。
3. 将新版本立即接受为当前正式正文。
4. 清除对应工作稿，并刷新章节、版本和字数数据。

## 边界与错误处理

- 正文为空时禁用操作，避免创建空正稿。
- 保存期间禁用重复提交。
- 任一步失败时保留已有正式正文和工作稿，界面显示具体错误。
- 版本接受沿用服务端现有的乐观并发校验；如章节已被其他操作更新，提示用户刷新后重试。

## 测试

新增端到端回归测试，覆盖手动工作稿保存为正稿后：

- 正式正文更新为工作稿内容；
- 历史版本增加一条手动正稿；
- 工作稿被清除。
