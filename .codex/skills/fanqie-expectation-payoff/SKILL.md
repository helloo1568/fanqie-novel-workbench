# 期待—阻力—兑现—状态结算

## R — 原文（Reading）

> 期待感在书里完成了，就是爽点；不必刻意写爽点，只需要完成期待感。
>
> — 《五行混沌经》作者访谈，第199—204行

## I — 方法论骨架（Interpretation）

爽点首先是一个具体期待：读者知道主角想得到什么、阻力是谁、成功意味着什么。铺垫要累积不公平、困难或信息差；爆发要由主角的选择与能力完成；见证者反应让结果可感；结算把胜负写入资源、身份、关系、认知或新责任。

同一种能力可以重复使用，但期待、代价、见证者和结果层级必须变化。若赢完以后下一章还能原样重来，前一场只是表演，没有成为故事。

## A1 — 资料集中的应用（Past Application）

### 案例 1：旁观者情绪反推爽点
- **问题**：马甲群像中的爽点容易只剩作者自嗨
- **方法论的使用**：作者先模拟反派制造憋屈，再从旁观者最想看到的释放反推行动和反馈
- **结论**：期待与见证反应同时设计
- **结果**：作品形成高在读和评分，但读者分层明显

### 案例 2：异常电费从怀疑到反击
- **问题**：小异常如何产生男频悬疑爽感
- **方法论的使用**：用事件、机制、情绪三层钩子蓄势，最终让反击回答前期问题
- **结论**：爽感来自答案与情绪释放的共同结算
- **结果**：平台选为高热案例，未披露量化数据

## A2 — 触发场景（Future Trigger）

### 用户会在什么情境下需要这个 skill？

1. 主角赢了但读者没感觉
2. 连续三次打脸结构相同
3. 高潮后身份、资源和关系都没变

### 语言信号

- “爽点不爽”
- “怎么蓄势和结算”
- “奖励落袋”
- “payoff / satisfaction loop”

- 与 fanqie-chapter-state-diff 的区别：本 skill 设计完整情绪闭环；后者只判断章前章后是否变化。
- 与 fanqie-character-conflict-network 的区别：人物网提供阻力动机；本 skill 安排兑现过程。

## E — 可执行步骤（Execution）

1. **写具体期待**
   - 动作：写读者等待的结果、成功标准和失败代价。
   - 完成标准：能用一个可回答问题表达，不用‘变爽/逆袭’空词。

2. **设计递进阻力**
   - 动作：至少安排一次成本、一次误判或关系压力，使主角必须选择。
   - 完成标准：阻力来自既有规则与人物欲望，不靠临时加难度。

3. **完成爆发与见证**
   - 动作：让主角用已建立能力/选择完成结果，并展示关键见证者的差异反应。
   - 完成标准：结果有正文证据；反应符合人物立场。

4. **写状态结算**
   - 动作：更新资源、身份、关系、认知、风险或责任，并由此生出下一期待。
   - 完成标准：至少一项不可逆变化进入记忆台账。

5. **做重复度检查**
   - 动作：与最近5—10章比较能力用法、阻力、见证、奖励和钩子。
   - 完成标准：至少三项不同；否则重构而非换名词。

## B — 边界（Boundary）

### 不要在以下情况使用

- 治愈、悲剧和情感文可用广义情绪兑现，不强行打脸。
- 不要让配角只负责震惊，也不要用重复反应替代结果。
- 平台成功案例没有证明某一爽点密度必然有效。

### 资料集警告的失败模式

- 重复打脸把人物写成道具。
- 高潮无铺垫且无后续影响。
- 中段重复任务却不改变局势。

### 证据局限

- 官方课程覆盖 2021—2026 年，市场、产品和平台规则必须按日期复核。
- 成功作者案例存在幸存者偏差，方法用于诊断和降低风险，不承诺流量或收益。
- 视频课程部分来自本地 ASR 转写，重要术语与硬规则需回查原始页面或多源证据。
