# 题材双轴与阅读承诺

## R — 原文（Reading）

> 选题要从主观与客观两方面判断；客观看覆盖面，还要具有延续性，并有悬念支撑。
>
> — 《做好选题，让新书“事半功倍”》00:00:39、00:03:24、00:08:35

## I — 方法论骨架（Interpretation）

先把“我想写”与“读者愿意长期读”分开检查。主观轴看作者的兴趣、经验、资料渠道与长期生产能力；客观轴看目标读者、核心欲望、同类差异、题材时效和持续冲突。两轴都通过后，再把书名、简介、开篇和长期正文写成同一份阅读承诺。

点子的新奇只能赢得第一次点击，支撑长篇的是它能否不断改变人物的目标、资源、关系和身份。包装不是独立营销层：包装说出的利益，正文必须尽快展示，并在不同阶段重复兑现。

## A1 — 资料集中的应用（Past Application）

### 案例 1：重度品类阅读者选择首书
- **问题**：新作者没有写作履历，不知道首本写什么
- **方法论的使用**：先完整阅读目标分类榜单，确认自己熟悉该品类的典型承诺和读者厌恶点
- **结论**：在阅读能力圈内做差异化，而不是追陌生热榜
- **结果**：编辑匿名案例披露首书收入二十万元，但存在选择偏差

### 案例 2：热门元素按厌恶点重组
- **问题**：多种豪门现言元素容易拼贴和伦理错位
- **方法论的使用**：作者先列目标读者反感的虐点，再约束男主行为与元素组合
- **结论**：包装与人物伦理共同服务同一体验
- **结果**：案例作品达到40万+在读；结果仍不能证明单一因果

## A2 — 触发场景（Future Trigger）

### 用户会在什么情境下需要这个 skill？

1. 准备开一本新书，但只有一个点子
2. 书名简介有点击感，正文却越写越像另一本书
3. 需要判断同人梗、系统或职业设定能否支撑长篇

### 语言信号

- “这个梗能写300章吗”
- “卖点到底是什么”
- “书名和正文对不上”
- “topic-market fit / book promise”

### 与相邻 skill 的区分

- 与 fanqie-story-engine-contract 的区别：本 skill 先决定卖什么、写不写；后者把已选题材变成可运行主线。
- 与 fanqie-genre-length-calibration 的区别：本 skill 做开书决策；后者提供题材和篇幅参数。

## E — 可执行步骤（Execution）

1. **写两轴证据表**
   - 动作：分别列出作者可持续能力、资料渠道、目标读者、核心欲望、同类差异和时效风险。
   - 完成标准：每轴至少3条可核验证据；缺证据项标记未知，不用形容词代替。

2. **做续航压力测试**
   - 动作：从核心梗推导至少3种不同冲突来源、3种阶段收益和2次主角目标升级。
   - 完成标准：能自然覆盖至少一个分卷；若只能产生同一种桥段，判定为开篇事件而非长篇题材。

3. **写一句阅读承诺**
   - 动作：用‘目标读者因为什么持续看主角获得什么体验’写一句话，并反查书名、简介、前三章和终局。
   - 完成标准：四处都能指向同一利益，且没有包装承诺正文不提供的内容。

4. **形成创作契约**
   - 动作：记录题材、主角目标、底层规则、不可变承诺和允许实验的部分。
   - 完成标准：核心项可锁定；未来修改必须显式重新确认。

## B — 边界（Boundary）

### 不要在以下情况使用

- 仅需要润色一个已确定书名时，不做完整选题评估。
- 不要把榜单热词、短期活动或作者收入个案当作需求证明。
- 平台课程存在幸存者偏差，双轴只能降低错配风险，不能保证流量。

### 资料集警告的失败模式

- 盲目追榜跟风，题材与作者能力不匹配。
- 包装承诺与正文错位。
- 单一噱头支撑不了长篇。

### 证据局限

- 官方课程覆盖 2021—2026 年，市场、产品和平台规则必须按日期复核。
- 成功作者案例存在幸存者偏差，方法用于诊断和降低风险，不承诺流量或收益。
- 视频课程部分来自本地 ASR 转写，重要术语与硬规则需回查原始页面或多源证据。
